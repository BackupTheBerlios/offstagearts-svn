Hi sthomps,

I read your message yesterday, and I said: it's a while I don't try to run iReport using JWS... why don't try now?

It was a pain, but finally with some modifications, I was able to succesfully run the last version of iR using JWS. Basically there are some problems related to the custom class loader used by iReport. To succesfully use JWS, you have to put in your user home directory a .java.policy file with the following content:

Code:


// this policy file should only be used for testing and not deployed

grant {

    permission java.security.AllPermission;

};



Really you can limit these permissions to all the applications started from a particular site.

Then you have to sign all the jars used by iReport (including ireport-1.2.6.jar).

I have attached a working jnlp.

This is fine to run iReport, but we have to solve 3 other problems:
1. in this way iReport is not able to find jars to compile reports (this problem is now fixed in the repository, I added a special argument "webstart" to activate a special classpath management when iReport is started using JWS
2. you have to "jarize" the templates (usually located in the templates directory) as described in the iReport manual
3. you have to "jarize" the plugins (usually located in the plugin directory) as described in the iReport manual

that's all
